#### 1. Клонирование репозитория
```bash
git clone <repository-url>
cd ProjectKanban
```

#### 2. Подготовка БД
```bash
createdb kanban_db
# или через pgAdmin/DBeaver
```

```bash
./gradlew clean build
java -jar build/libs/kanban-*.jar
```

#### 4. Frontend запуск
```bash
cd src/frontend/frontend
npm install
npm run dev
```
Приложение будет доступно на: `http://localhost:5173`

## 🏛️ Архитектура проекта

### Общая архитектура
- **Backend:** Spring Boot 3, GraphQL API, REST для файлов, JWT аутентификация, Spring Data JPA, асинхронная отправка email, шедулер для просроченных задач, кэширование (Spring Cache).
- **Frontend:** React 18, Apollo Client (GraphQL), React Router, CSS Modules (разбиты по компонентам), Vite.
- **База данных:** PostgreSQL с нормализованной схемой (проекты, участники, группы, задачи, подзадачи, теги, вложения).

### Безопасность
- JWT токен хранится в `localStorage`.
- Аутентификация через Spring Security + JwtAuthenticationFilter.
- Доступ к проектам проверяется по ролям (OWNER, ADMIN, MEMBER, VIEWER) и членству в проекте.
- Защита от ботов при регистрации: Cloudflare Turnstile.

### Ключевые особенности реализации

#### Бэкенд
1. **Отложенные email‑уведомления об изменениях задачи**  
   В `TaskService` изменения накапливаются в течение 2 секунд, после чего отправляется одно письмо со списком изменений (уменьшает количество писем при быстрых правках).

2. **Загрузка задач с тегами без N+1**  
   Сначала загружаются задачи с исполнителями (без тегов), затем отдельным запросом подгружаются теги для всех ID задач.

3. **Управление правами**
   - На уровне проекта: OWNER, ADMIN, MEMBER, VIEWER.
   - На уровне подгруппы: LEADER, MEMBER.
   - На бэкенде проверки через `ProjectMember` и `SubgroupMember`.

4. **Подзадачи** реализованы через поле `parentTaskId` (самоссылка). При отображении рекурсивно раскрываются.

5. **Асинхронная отправка писем** не блокирует ответ GraphQL.

#### Фронтенд
- **Авторизация:** `AuthContext` управляет состоянием пользователя и токеном.
- **Кэширование Apollo:** кэш InMemoryCache с персистенцией в `localStorage`.
- **Маршрутизация:** защищённые маршруты (`PrivateRoute`) проверяют авторизацию и подтверждение email.
- **Компонентная структура:** логически разделена на панели, модальные окна, списки.
- **Адаптивность:** CSS media queries для мобильных устройств, горизонтальный скролл на панели групп, выезжающие модальные окна.

## 📊 Управление проектами

### Создание проекта
1. На главной странице нажмите "Создать проект"
2. Введите название
3. Нажмите "Создать"

### Управление участниками
1. Перейдите в "Настройки проекта" (доступно для OWNER/ADMIN/MEMBER)
2. В разделе "Участники":
   - **Добавить**: введите email и выберите роль
   - **Изменить роль**: используйте dropdown рядом с участником
   - **Удалить**: нажмите кнопку "Удалить"

### Роли доступа в проекте
| Роль | Права |
|------|-------|
| **Владелец** | Полный доступ, управление проектом, удаление |
| **Админ** | Управление участниками, группами, настройками |
| **Участник** | Создание и редактирование задач, просмотр проекта |
| **Наблюдатель** | Только просмотр, без прав на редактирование |

### Уведомления
- **Глобальные:** настройка в личном кабинете (`/account`).
- **По проекту:** в настройках проекта можно переопределить глобальные настройки.
- **По задаче:** уведомления при назначении, изменении, добавлении файлов, просрочке (только если задача не завершена).

## 👥 Управление группами и ролями

### Что такое группа?
Группа (подгруппа) – это контейнер для задач внутри проекта. Позволяет разделять работу по направлениям, спринтам, командам.

### Создание группы
1. На канбан‑доске нажмите кнопку **«+»** рядом с заголовком «Группы»
2. Введите название группы
3. Группа создаётся, и вы становитесь её **лидером**

### Роли внутри группы

| Роль | Права |
|------|-------|
| **Лидер** | Управление участниками группы, настройка названия, удаление группы, добавление/удаление задач |
| **Участник** | Создание и редактирование задач в группе, просмотр всех задач группы |

> **Примечание:** Назначение участников в группу возможно только из числа участников проекта (исключая наблюдателей). Лидер группы может добавить участника и при необходимости повысить его до лидера или понизить до участника.

### Управление участниками группы
1. Откройте настройки группы (шестерёнка рядом с названием)
2. В разделе «Участники» вы увидите список текущих членов
3. **Добавить** – выберите участника проекта из выпадающего списка и его будущую роль
4. **Изменить роль** – используйте селектор напротив участника (доступно только лидерам)
5. **Удалить** – удалить участника из группы

### Особенности
- Лидер группы может выполнять все действия, доступные участнику.
- Участник группы не может управлять её составом или удалять группу.
- Права на группу не влияют на права в проекте (например, наблюдатель проекта не может быть добавлен в группу).

## 📝 Управление задачами

### Структура задачи
```
Основная задача
├── Подзадача уровня 1
│   ├── Подзадача уровня 2
│   │   └── Подзадача уровня 3
│   └── ...
├── Подзадача уровня 1
└── ...
```

### Создание задачи
1. Нажмите кнопку "+ Новая задача"
2. Заполните поля:
   - **Название** (обязательно)
   - **Описание** (опционально)
   - **Приоритет** (1-3)
   - **Дедлайн** (опционально, можно выбрать «Сейчас»)
   - **Исполнители** (можно выбрать несколько)
3. Нажмите "Создать"

### Операции с задачами
- **Перемещение**: Drag-and-drop между колонками статусов
- **Редактирование**: Клик по задаче → редактирование в модале
- **Добавление подзадачи**: Правый клик → "Добавить подзадачу"
- **Удаление**: Правый клик → "Удалить задачу"
- **Теги**: Правый клик → выбор или создание тега
- **Вложения**: Добавьте файлы в модале задачи

### Фильтрация и сортировка
**Фильтры:**
- По приоритету (Низкая, Средняя, Высокая)
- По диапазону дат (с ... по ...)
- По исполнителю
- По тегам (множественный выбор)

**Сортировка:**
- По дедлайну (возрастание/убывание)
- По приоритету (возрастание/убывание)
- По создателю (A-Z/Z-A)

### Цветовые индикаторы
- 🔴 **Красный**: Просрочено
- 🟠 **Оранжевый**: Менее суток до дедлайна
- 🟡 **Желтый**: Менее 3 дней до дедлайна
- 🔵 **Синий**: Нормально

## 💾 Экспорт и импорт

### Экспорт задачи

**Как использовать:**
1. Откройте канбан-доску
2. Щелкните правой кнопкой мыши на задаче
3. Выберите "Экспортировать задачу"
4. Файл загрузится как: `task_{id}_{название}.json`

**Пример файла:**
```json
{
  "title": "Важная задача",
  "description": "Описание задачи",
  "dueDate": "2025-12-31T23:59:59.000Z",
  "value": 3,
  "status": "IN_PROGRESS",
  "subTasks": [
    {
      "title": "Подзадача 1",
      "description": null,
      "dueDate": null,
      "value": 2,
      "status": "TODO",
      "subTasks": []
    }
  ]
}
```

**Примеры названий файлов:**
- `task_123_vazhnaya_zadacha.json`
- `task_45_fix_bug_123.json`

### Экспорт группы

**Как использовать:**
1. Перейдите на канбан-доску
2. Нажмите иконку ⚙️ на группе
3. Нажмите "Экспорт группы"
4. Файл загрузится как: `group_{id}_{название}.zip`

**Содержимое ZIP:**
```
group_{id}_{name}.zip
├── group.json (metadata и все задачи)
└── attachments/
    └── task_{id}/ (вложения по задачам)
```

### Экспорт проекта

**Как использовать:**
1. Перейдите в "Настройки проекта" (требуются права ADMIN/OWNER)
2. Нажмите кнопку "Экспорт проекта"
3. Файл загрузится как: `project_{id}_{название}.zip`

**Содержимое ZIP:**
```
project_{id}_{name}.zip
├── project.json (структура, все группы и задачи)
└── attachments/ (все вложения)
```

### Импорт задач

**Как использовать:**
1. Откройте нужную группу на канбан-доске
2. Нажмите кнопку "Импорт" в панели инструментов
3. Выберите JSON файл с задачей(ами)
4. Подтвердите импорт
5. Задачи будут добавлены в группу

**Поддерживаемые форматы:** одиночная задача или массив задач.

**Важные особенности:**
- Иерархия подзадач сохраняется полностью
- Статусы автоматически нормализуются (0→TODO, 1→IN_PROGRESS, 2→REVIEW)
- По умолчанию: приоритет=2 (Средняя), статус=TODO
- Импорт не удаляет существующие задачи

### Импорт группы

**Как использовать:**
1. Перейдите на канбан-доску проекта
2. Нажмите иконку ⬆️ в панели групп
3. Выберите ZIP файл экспортированной группы
4. Группа будет создана с названием "оригинальное (imported)"
5. Все вложения будут восстановлены

**Поддерживаемые форматы:** ZIP файл от экспорта группы

### Импорт проекта

**Как использовать:**
1. На странице списка проектов нажмите кнопку "Импорт проекта"
2. Выберите ZIP файл экспортированного проекта
3. Создаётся новый проект с названием "оригинальное (imported)", текущий пользователь становится владельцем

## 🔌 API документация

### GraphQL API
**Endpoint:** `POST /graphql`

**Основные query и mutation:**

```graphql
# Получить проект со всеми деталями
query GetProject($projectId: ID!) {
  project(id: $projectId) {
    id name
    owner { id fullName email }
    members { id userId role user { fullName email } }
    subgroups { id name tasks { id title status value dueDate } }
  }
}

# Создать задачу
mutation CreateTask($subgroupId: ID!, $title: String!, $assigneeIds: [ID!]) {
  createTask(subgroupId: $subgroupId, title: $title, assigneeIds: $assigneeIds) {
    id title status
  }
}
```

### REST API Endpoints

#### Экспорт
```
GET  /api/export/project/{projectId}     (требуется членство в проекте)
GET  /api/export/subgroup/{subgroupId}   (требуется членство в проекте)
```

#### Импорт
```
POST /api/import/project                 (создаёт новый проект, текущий пользователь – владелец)
POST /api/import/subgroup/{projectId}    (добавляет группу в существующий проект)
```

#### Файлы
```
POST   /api/files/upload/{taskId}        (загрузка файла, multipart)
GET    /api/files/{id}                   (скачивание файла)
DELETE /api/files/{id}                   (удаление файла)
```

### Коды ответов
| Код | Значение |
|-----|----------|
| 200 | OK |
| 201 | Created |
| 400 | Bad Request |
| 401 | Unauthorized |
| 403 | Forbidden |
| 404 | Not Found |
| 500 | Server Error |

## 🐛 Устранение неполадок

### Backend не запускается – Connection refused
```bash
# Измените порт
server.port=8081
```

### БД не подключается – password authentication failed
```bash
# Проверьте учётные данные в application.properties
psql -U postgres -d kanban_db
```

### Frontend не видит API – CORS ошибка
- Убедитесь, что бэкенд запущен на правильном порту.
- Проверьте `VITE_API_URL` в .env.
- Проверьте `cors.allowed-origins` в настройках бэкенда.

### Импорт/Экспорт не работает – 403 Forbidden
- JWT токен мог истечь – выполните повторный вход.
- Убедитесь, что у вас есть права на проект (участник, не наблюдатель).

### Задачи не импортируются – Invalid enum value
- В JSON статус должен быть: TODO, IN_PROGRESS или REVIEW (регистр не важен).
- Цифровые значения (0,1,2) также поддерживаются.
